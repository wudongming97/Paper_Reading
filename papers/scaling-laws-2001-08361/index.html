Scaling Laws for Neural Language Models — 语言模型越大，数据用得越「省」

这篇 2020 年的 OpenAI 论文用大量实验说明：语言模型的交叉熵损失随参数量 N、训练 token 数 D、算力 C 呈光滑的幂律下降，跨度超过七个数量级。最反直觉的结论之一：在固定算力下，最优策略往往是训很大的模型、用相对少的数据、在还没收敛时就停——大模型比小模型更「样本高效」。

Setting the stage

  • 语言模型在做什么。 给定前文，预测下一个词（token）。训练目标是最小化交叉熵损失（cross-entropy loss）——可理解为「猜下一个词有多准」；损失越低，模型越好。
  • Transformer 是当时的标配架构。 一种用自注意力（self-attention）处理整段上下文的神经网络，GPT 系列即属此类。本文主要在 WebText2 文本上训练仅解码器的 Transformer。
  • 「缩放」（scaling）指同时放大三样东西。 非嵌入参数量 N、数据集大小 D（token 数）、训练算力 C。论文问：损失如何随这三者变化？给定固定算力，该怎么分配？
  • 算力 C 的估算。 近似 \(C \approx 6N \times B \times S\)：每个 token 约 6N 次浮点运算（前向+反向），B 为 batch 大小，S 为参数更新步数。文中常用 PF-day（\(10^{15}\) 次运算 × 一天）作单位。

The problem

  • 训练大模型很贵，但没人有统一「配方」。 2019–2020 年 GPT-2 等已很强，但模型该多大、数据要多少、训多久，多靠经验而非可预测规律。
  • 架构超参太多，难以单独优化。 层数、宽度、注意力头数等组合爆炸；若性能强烈依赖形状，规划算力预算几乎不可能。
  • 过拟合与算力浪费难以量化。 模型相对数据过大时会过拟合；相对算力过小则欠拟合。需要可外推的公式，而不是每次重新扫一遍网格。
  • 跨数据集表现是否可预测？ 在 WebText 上训好的模型，换到 Wikipedia、书籍等分布，性能如何随规模变化？

The core idea

  • 损失随规模呈幂律，且跨多个数量级成立。 当 N、D、C 中某一因素成为瓶颈时，测试损失 \(L\) 近似 \(L \propto X^{-\alpha_X}\)，在论文拟合范围内未见明显偏离（上限处最终必趋于零损失的平台，本文未触及）。
  • 形状（宽/深）远不如规模重要。 固定非嵌入参数量 N 后，层数、头数、FFN 比例等在很宽范围内只带来百分之几的损失波动（见图 5）。
  • 大模型更样本高效。 达到同样损失，大模型需要更少训练步、更少 token（见图 2、图 4）。
  • 固定算力下的最优策略：大模型 + 早停。 不把小模型训到收敛，而是用更大模型在远未收敛时停止——这在算力效率上更优。

How it works

  • 三条基本幂律（各自单独成为瓶颈时）。 充分数据、训到收敛：\(L(N) = (N_c/N)^{\alpha_N}\)，\(N_c \sim 8.8\times 10^{13}\)。数据有限、大模型早停：\(L(D) = (D_c/D)^{\alpha_D}\)，\(D_c \sim 5.4\times 10^{13}\) token。算力有限、模型与 batch 最优：\(L(C_{\min}) = (C_c^{\min}/C_{\min})^{\alpha_C^{\min}}\)，\(C_c^{\min} \sim 3.1\times 10^8\) PF-day。
  • 联合公式：N 与 D 如何配对。 同时依赖模型与数据时，\(L(N,D) = \big[(N_c/N)^{\alpha_N} + (D_c/D)^{\alpha_D}\big]^{\alpha_D}\)（式 1.5）。含义：N 与 D 应同步放大；只增大其一会进入收益递减。经验上模型增大 8×，数据约需增大 5× 才能避免过拟合惩罚（\(N^{0.74}/D\) 比例）。
  • 训练曲线也可拟合。 无限数据、有限步数 S：初期 transient 后，\(L(N,S) \approx (N_c/N)^{\alpha_N} + (S_c/S_{\min}(S))^{\alpha_S}\)，\(\alpha_S \approx 0.76\)。可用早期曲线外推长期损失。
  • 临界 batch 大小。 \(B_{\mathrm{crit}}(L) = B_* / L^{1/\alpha_B}\)，\(B_* \sim 2\times 10^8\) token，\(\alpha_B \approx 0.21\)。在临界 batch 附近可在时间与算力效率间取得较好折中。

What's new

  • 相对早期「宽 vs 深」经验法则。 此前多关注手工调形状；本文系统证明在固定 N 下形状影响极小，应把预算放在放大 N、D、C。
  • 相对 Hestness 等算力缩放工作。 更早工作在较小规模、不同任务上发现幂律；本文在语言建模上跨度更大（N 跨 6+ 数量级、C 跨 8+ 数量级），并给出 N–D–C 联合分配。
  • 相对「训到收敛」的习惯。 社区常把小模型训很久；本文指出算力最优路径是 \(N \propto C^{0.73}\)、batch \(B \propto C^{0.24}\)、步数 \(S \propto C^{0.03}\)——算力增加时，绝大部分应花在更大模型上，数据只需次线性增加（\(D \sim C^{0.27}\)）。
  • 迁移行为简单。 在 WebText2 外的 Books、Wikipedia 等分布上，损失随 N 的幂律斜率几乎相同；分布偏移带来近似常数损失偏移，与训练多久、是否接近收敛关系不大。

常见做法

小模型 + 训到收敛
算力花在大量步数上；样本效率低；在固定 C 下并非最优

算力最优（Chinchilla 之前的主流解读）

大模型 + 早停 + 适中 D
同样 C 下损失更低；数据需求随 C 增长很慢（\(\sim C^{0.27}\)）

Results

  • 幂律跨 7+ 数量级。 损失对 N、D、\(C_{\min}\) 的光滑幂律关系在实验范围内稳定，未见明显拐点（见论文 Figure 1）。
  • 参数量翻倍的效果。 \(\alpha_N \approx 0.076\) → 损失约乘以 \(2^{-0.076} \approx 0.95\)，即约 5% 相对改善——看似不大，但跨多个翻倍累积显著。
  • 架构鲁棒性。 宽高比变化 40×，损失仅差约 3%（与 GPT-2 规模相当的 (6,4288) vs (48,1600)）。
  • Transformer 优于 LSTM 在长上下文。 同参数量下，LSTM 在上下文前段尚可，后段明显落后；Transformer 在整个 1024 token 窗口持续改进（Figure 7）。

Limitations

  • 数据集与分词器绑定。 \(N_c, D_c\) 等常数依赖 WebText2 与 BPE 词表（50257）；换语料/词表需重新拟合，不宜当作普适物理常数。
  • 上限必有平台。 损失不能低于零；论文未观察到上界偏离，但极高算力/规模下幂律终将失效。
  • 硬件现实约束。 最优方案常因内存无法训练「足够大」的模型，实践中仍常训小于算力最优的模型更久。
  • 后续工作修正了 N–D 平衡。 2022 年 Chinchilla 等工作指出，在更新算力预算下，数据规模应与模型更均衡地缩放；Kaplan 定律仍描述现象，但「最优分配」需结合时代硬件与目标更新（见 Influence）。

Influence

  • GPT-3 与大规模预训练路线图。 为「继续放大模型」提供了可量化依据；直接影响 2020 年前后工业界 scaling 叙事。
  • Chinchilla（Hoffmann et al., 2022）。 在类似框架下重新拟合，强调同等算力下应增加数据而非一味放大 N——可看作对本论文「\(D \sim C^{0.27}\)」结论的修正与延伸。
