Human-to-Robot Motion Retargeting — 把人体动作搬到人形机器人上

这篇（示例）论文的核心思路是：先把人体动作「翻译」成机器人能执行的关节轨迹，再用强化学习在真机上跟踪这些轨迹，从而缩小仿真与现实的差距。

Setting the stage

  • 人形机器人控制。 让双足机器人像人一样走、平衡、伸手——通常用神经网络策略，而不是手写控制器。
  • 策略（policy）。 从传感器读数到电机指令的映射；现代做法多用强化学习（RL）在仿真里训练，再部署到硬件。
  • Sim-to-real。 在物理仿真里训练（快、安全），再上到真机（慢、易摔）。多数机器人 RL 论文都在解决「仿真和真机不一致」。

The problem

  • 仿真里好用的策略，真机上常崩。 仿真器的电机动力学和真实硬件有偏差。
  • 人体动作数据很多，但不能直接照搬。 动捕（mocap）记录的是人体骨架；29 关节人形机器人几何不同，硬抄会不稳。
  • 老的重定向只做运动学。 只给关节角度序列，不保证真机电机跟得上、能平衡。

The core idea

  • 拆成两步。 人体→机器人是几何问题；真机执行是另一个 RL 问题，两阶段不必共用一个模型。
  • 用 RL 补上物理缺口。 策略在跟踪重定向命令的同时，学会电机极限、接触和平衡。

How it works

  • 阶段 1：运动学重定向。 把人体姿态映射到机器人关节空间，满足可达性与姿态约束。
  • 阶段 2：RL 跟踪策略。 在仿真里奖励跟踪误差小、能耗低、不摔倒；再 sim-to-real 微调。
  • 奖励设计。 跟踪项 + 正则项 + 平衡惩罚，避免只会「抖」着跟轨迹。

What's new

  • 对比纯运动学管线。 老方法到真机就断；这里 RL 专门负责「能执行」。
  • 对比端到端模仿。 不强迫一个网络同时学几何和物理，分工更清晰。

Results

  • 示例数字。 在标准人形平台上，跟踪成功率从 ~60% 提到 ~90%（示意，非真实论文数据）。

Limitations

  • 仍依赖仿真质量。 RL 阶段若仿真偏差大，真机仍会漂。
  • 极端动作难泛化。 跳跃、翻滚等超出训练分布的动作未覆盖。
